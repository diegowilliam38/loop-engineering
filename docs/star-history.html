GitHub access token

Star-history reads star data from the GitHub API. GitHub now restricts starred data to a repository's own admins and collaborators, so star history is only available for repos you own or collaborate on — and you'll need an access token that can read them.

Create a fine-grained token with only Metadata → Read-only permission (all the stargazers endpoint needs), or a classic token with the public_repo scope, then paste it below. More details.

A token with no scopes no longer works, even for your own repositories.

  • Fine-grained (recommended): Repository access → this repo (or All repositories). Permissions → Metadata → Read-only only.
  • Classic: tick public_repo — also works for repos you collaborate on.

Your token is stored only in this browser's localStorage — it is never sent to Loop Engineering servers. Chart requests go directly from your browser to GitHub's API.
